FAQ

Common questions

What is a "vertical"?
A vertical is a research topic category — for example, E-Commerce Logistics, Digital Marketing, Consumer Psychology, Health & Wellness, or Finance. PaperBrief reads each paper's content and routes it to the correct vertical automatically. Your digest is then organized by vertical so findings from unrelated fields never get mixed together. On the Monthly plan you specify one vertical; Annual gives you three; Enterprise is unlimited.
Do I need an Anthropic API key to use PaperBrief?
No. PaperBrief's heuristic extraction engine runs with zero external API keys — it parses PDFs locally, pulls key sentences and statistics, and writes a structured brief without any LLM. When you set ANTHROPIC_API_KEY in your .env file, the agent upgrades to Claude-written synthesis for richer, more nuanced takeaways. The API key is opt-in, not required.
How many papers can I submit at once?
There is no hard limit on the number of PDFs you can drop into data/pb_pdfs/ at one time. The agent processes all unread papers on each run. For very large batches (50+ papers) the agent may spread processing across two consecutive cron cycles to avoid overloading the email digest with too many findings at once. Each processed paper is tracked so it is never re-summarized on a future run.
Can I add my own verticals?
Yes. Verticals are defined as keyword clusters in paperbrief/tools.py. You can add a new entry to the VERTICALS dict with the name and a list of topic keywords — the routing logic scores each paper against every vertical and assigns it to the highest match. On Enterprise, custom vertical configuration is included in the setup call. On Monthly and Annual, you can edit the config file yourself following the inline comments.
