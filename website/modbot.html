FAQ

Common questions

How do I submit comments for moderation?
Drop a JSON batch file into data/cm_inputs/{slug}.json with your account name, platform, and a list of comment strings. Run ModBot and the classification report lands in data/cm_outputs/{slug}.json with every comment labeled, scored, and replied. No API keys required for the heuristic rules — the agent processes everything locally.
What triggers an auto-hide?
Six pattern classes fire an automatic hide at very high confidence: profanity and slurs, crypto/investment DM solicitations, WhatsApp or Telegram phone-number drops, posts containing three or more URLs, mass-character-repetition spam (e.g., "aaaaaa"), and known scam-tell phrases. These rules never escalate to the AI model — they are hard-coded for speed and reliability at 99–100% confidence.
Can I customize the reply templates?
Yes. The reply templates for positive engagement, questions, and flagged comments live in modbot/tools.py as a plain Python dictionary — edit the strings directly. No retraining, no config file format to learn. Each template supports a {account} and {platform} placeholder so replies feel native to the brand voice.
Does this work with comment management tools like Sprout Social?
ModBot outputs a structured JSON report — one object per comment with decision, confidence, reason, and suggested_reply fields. You can pipe that JSON into Sprout Social, Hootsuite, or any tool that accepts JSON-based automation via Zapier, Make, or a custom webhook. ModBot does not push actions directly to platforms, so your existing moderation workflow stays in control.
