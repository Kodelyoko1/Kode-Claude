FAQ

Common questions

What file formats does ProofBot accept?
Plain text (.txt) and Markdown (.md). Drop files into data/pf_inputs/ or let auto-ingest pull from SEOWriter, ShowNotes, or PaperBrief outputs. The agent processes all unprocessed files each run.
Is this the same as Grammarly?
Similar spirit, different stack. ProofBot is a local script that outputs a structured issues table — no browser extension, no cloud subscription, no tracking. It integrates directly into the agent pipeline so articles are proofed automatically as they're generated. The homophone heuristics also catch patterns Grammarly regularly misses in conversational copy.
Does it rewrite my content?
No. The "cleaned copy" block only applies deterministic safe fixes: whitespace normalization, double-space removal, and repeated-word removal. All subjective grammar suggestions appear in the issues table with the suggested correction — you decide which to apply. ProofBot never silently rewrites phrasing.
What if LanguageTool's API is slow or rate-limited?
The heuristic rules (homophones, repeats, whitespace, headings) run fully offline and are never rate-limited. LanguageTool's free tier allows ~20 requests/minute — for large batches the agent paces requests automatically. If the API is unavailable, you still get the heuristic report; the LanguageTool section shows a "skipped — API unavailable" note.
