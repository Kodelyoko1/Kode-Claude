Four Systems

What Batman checks every night

🪵

Log Scanner

Reads the last N run log files and extracts per-agent failure lines and Python tracebacks. Each failure is attributed to the correct agent by reading the === Agent: name === header blocks that every run_*_auto.py writes.

🔍

JSON Integrity Checker

Try-loads every data/*.json to catch truncated or corrupted files before they take an agent offline mid-cycle. Files modified in the last 60 seconds are skipped (live-write guard) to avoid false alarms on in-progress writes.

⏰

Stale Agent Detection

Compares each agent's last_success timestamp to configurable stale thresholds. Weekly-cycle agents automatically get a 5× multiplier so they don't false-alarm between their scheduled runs.

🔧

Auto-Repair (BATMAN_LIVE=1)

Quarantines broken files to data/.batman_quarantine/<ts>-<agent>-<file> and restores known empty schemas (agent_metrics.json → {}, leads.json → {}, etc.). Every action is logged in the digest.

FAQ

Common questions

Does Batman require BATMAN_LIVE=1 to do anything useful?
No. Dry-run mode (the default) still scans logs, checks JSON integrity, detects stale agents, and emails the full digest. BATMAN_LIVE=1 only enables the file-quarantine and schema-restoration actions — all the visibility is free of any destructive operations until you're ready to enable live mode.
How does Batman know which agent caused a failure?
It reads run log files and looks for === Agent: <name> === header lines above each traceback. Every run_*_auto.py writes one of these headers at the start of its cycle, so failures can be attributed even in a single unified log file covering all 14 agents.
What happens if Batman itself crashes?
Batman is wrapped with run_with_healing() like every other agent. If Batman fails 3 consecutive cycles, the owner gets an escalation email from the self-healing module — same as any other agent in the fleet. The supervisor supervises itself.
Can I add my own agents to the fleet?
Yes. Batman discovers agents automatically from run log headers. Any new run_*_auto.py that follows the standard logging pattern (writes the === Agent: name === header, appends to the shared log directory) appears in the next digest with no configuration changes needed.
